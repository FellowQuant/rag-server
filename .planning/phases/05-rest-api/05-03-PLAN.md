  <action>
Create scripts/verify_api.py. The script uses httpx (already in the project's virtualenv via fastapi dependencies) to make synchronous HTTP requests against a running server at BASE_URL = "http://localhost:8001" (configurable via BASE_URL env var).

Script structure:
- Define `BASE_URL` from env or default
- Define a `results: list[tuple[str, bool, str]]` list ā€” (test_name, passed, detail)
- Define helper `check(name, cond, detail="")` that appends to results and prints PASS/FAIL inline
- Run all checks sequentially using `httpx.Client(timeout=10.0)`
- At the end, print summary: "N/M checks passed" and exit with code 0 if all pass, 1 if any fail

Checks to implement (in order):

**Health check (API infrastructure):**
1. `GET /health` ā†’ status 200, body `{"status": "ok"}`

**URL versioning ā€” old paths return 404 (hard cut, API-01 through API-06 prefix requirement):**
2. `GET /documents` ā†’ status 404 (old path gone)
3. `GET /ask` ā†’ status 405 or 404 (old path gone ā€” 405 Method Not Allowed also acceptable since POST-only routes return 405 on GET)

**CORS headers (middleware requirement):**
4. `GET /api/v1/documents` ā†’ status 200 AND response header `access-control-allow-origin` == `*`

**Document list endpoint (API-02):**
5. `GET /api/v1/documents` ā†’ status 200, body has `documents` list and `total` int

**RFC 7807 error shape ā€” 422 (error handler requirement):**
6. `POST /api/v1/retrieve` with body `{}` (missing required `query` field) ā†’ status 422, body has keys `type`, `title`, `status`, `detail` (RFC 7807 shape, NOT `{"detail": [...]}` list format)

**RFC 7807 error shape ā€” 404:**
7. `GET /api/v1/documents/nonexistent-id-00000` ā†’ status 404, body has keys `type`, `title`, `status`, `detail`

**Retrieve endpoint ā€” global search (API-04):**
8. `POST /api/v1/retrieve` with body `{"query": "financial risk management", "top_k": 3}` ā†’ status 200, body has keys `query`, `results` (list), `total_candidates` (int)

**Retrieve endpoint ā€” with document_ids filter (API-04 filter requirement):**
9. `POST /api/v1/retrieve` with body `{"query": "test", "top_k": 5, "document_ids": []}` ā†’ status 200 (empty document_ids is a valid request ā€” returns empty results gracefully)

**Retrieve endpoint ā€” with min_score (API-04 threshold requirement):**
10. `POST /api/v1/retrieve` with body `{"query": "test", "top_k": 5, "min_score": 0.99}` ā†’ status 200 (high threshold likely returns empty results, but must not error)

**ChunkResultItem shape check (API-04 ā€” when corpus has data):**
11. `POST /api/v1/retrieve` with body `{"query": "test", "top_k": 1}` ā†’ if results list is non-empty, verify first result has fields: `chunk_id`, `document_id`, `chunk_index`, `content`, `source_filename`, `chunk_type`, `bm25_score`, `dense_score`, `sparse_score`, `rrf_score`, `reranker_score`. If results is empty (fresh server with no documents), skip this check and note "skipped (no documents indexed)".

**Ask endpoint availability (API-05):**
12. `POST /api/v1/ask` with body `{"query": "test", "top_k": 1}` ā†’ status 200 (server responds, even if LLM provider is unavailable ā€” a 500 means /ask is reachable but LLM failed, still counts as endpoint present). Accept 200 or 500 as "endpoint exists"; only 404 is a failure.

**Upload size limit ā€” 413 (middleware requirement):**
13. `POST /api/v1/documents` with header `Content-Length: 200000000` (200MB, exceeds 100MB limit) and minimal body ā†’ status 413, body has `type`, `title`, `status`, `detail` keys (RFC 7807 from UploadSizeLimitMiddleware). Note: send a small actual body ā€” the middleware checks the Content-Length HEADER value, not actual body length.

Print format for each check:
```
[PASS] GET /health ā†’ 200
[FAIL] GET /documents ā†’ expected 404, got 200
```

Final summary:

```

```

Exit code 0 on full pass, 1 on any failure.

httpx allows overriding Content-Length header explicitly.
